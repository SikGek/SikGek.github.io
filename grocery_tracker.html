Computer Vision based Grocery and Fridge Tracker

UCSD

Along with other engineering students, we set out to build a computer vision based grocery and fridge tracker using a single board computer (SBC). We began out using a Raspberry Pi4, as it was the most accessible and easiest to use UI, however, we intended to switch this out for an ESP32 if it ever went into production.

We concluded that by using a 720p camera and a simple CNN for computer vision, the user can place the device on top of their fridge to easily detect what is going in and out of the fridge by comparing pixel values of the detected object in two consecutive shots. After achieving this, we created a SQL database in a local server along with a fully functioning and connected app that would update the contents of the fridge live. We wished to be able to turn any fridge into a smart fridge that you could check on your phone. Similar to the object retreival vehicle I was not aware of any methods to significantly increase speed of deep learning computation on an SBC, which is why the object is inserted into the fridge at a slower pace. I believe, if I were to revist this idea, this could easily work in real time on the SBC without the use of server computation.
